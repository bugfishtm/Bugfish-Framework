PHP Debug Class [x_class_debug]Documentations

This is the Documentation for the Bugfish Framework!



Documentation

The x_class_debug class provides debugging and benchmarking functionalities for PHP applications. It includes methods to check for required PHP modules, monitor memory usage and limits, track CPU load, determine upload file size limits, and measure script execution time. Additionally, it offers a method to generate error screens for maintenance messages and missing modules. This class is useful for debugging, ensuring module availability, and benchmarking code performance.

Class Functions

Below a list of functions and explanations on how to use them.

Function Description
error_screen($text) Display Error Screen with Text
required_php_modules($array = array(), $errorscreen = false) Check if Array is active all modules, if not return array with inactive modules
required_php_module($name, $errorscreen = false) Check if required PHP Module is active
php_modules() Get Current Loaded PHP Modules in Array
memory_usage() Get Current Script Memory Usage
memory_limit() Get Current Value of ini Setting
cpu_load() Get Current CPU Load
timer() Get Current Timer for Site Loading Time
upload_max_filesize() Get Current Value of ini Setting

Example Implementation

Below you can see one example implementation of this class. It does only show a short introduction on how to initialize it.

// Include or require the file that contains the x_class_debug class

// Create an instance of the x_class_debug class
$debug = new x_class_debug();

// Check if a required PHP module is available
$requiredModule = 'mysqli'; // Replace with the module you need
if (!$debug->required_php_module($requiredModule, true)) {
// Display an error screen if the module is missing
// The 'true' parameter tells the class to display an error screen
}

// Check for multiple required PHP modules
$requiredModules = ['pdo', 'curl', 'gd']; // Replace with the modules you need
$missingModules = $debug->required_php_modules($requiredModules, true);
if (count($missingModules) > 0) {
// Display an error screen if any of the required modules are missing
// The 'true' parameter tells the class to display an error screen
}

// Check memory usage and limit
$memoryUsage = $debug->memory_usage();
$memoryLimit = $debug->memory_limit();
echo "Memory Usage: $memoryUsage
";
echo "Memory Limit: $memoryLimit
";

// Check CPU load
$cpuLoad = $debug->cpu_load();
echo "CPU Load: $cpuLoad
";

// Check upload max filesize
$maxFilesize = $debug->upload_max_filesize();
echo "Max Upload Filesize: $maxFilesize
";

// Start a timer
$debug->timer();

// Perform some operations that you want to measure

// Get the elapsed time from the timer
$elapsedTime = $debug->timer();
echo "Elapsed Time: $elapsedTime seconds
";

// Display an error screen with a custom message
$errorMessage = "Something went wrong!";
$debug->error_screen($errorMessage);

// Note: In a real application, you may want to customize the error messages and handling as needed.
