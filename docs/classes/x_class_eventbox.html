PHP Eventbox Class [x_class_eventbox]Documentations

This is the Documentation for the Bugfish Framework!



Documentation

The x_class_eventbox class in PHP provides a convenient way to manage and display various types of messages and notifications on a web page. This class simplifies the process of adding, displaying, and customizing messages, allowing developers to create user-friendly event boxes with ease.

⚠️ Uses PHP Sessions


Class Funtions

Below you can see a list and explanations about this classes functionalities!

Function Name Description Parameters
`__construct` Initializes an instance of the class. - `$cookie` (optional): A custom cookie name to distinguish event boxes (defaults to an empty string).
Properties - `private $cookie` - Stores the custom cookie name provided during initialization.
`ok` Adds an "OK" type message to the event box. - `$text`: The text message to display.
`warning` Adds a "Warning" type message to the event box. - `$text`: The text message to display.
`error` Adds an "Error" type message to the event box. - `$text`: The text message to display.
`info` Adds an "Info" type message to the event box. - `$text`: The text message to display.
`add` Adds a custom message to the event box. - `$text`: The text message to display. <br> - `$type`: The type of message (e.g., "ok", "warning", "error", "info").
`get` Retrieves the current array of messages in the event box. None
`override` Overrides all current messages in the event box with a single message. - `$text`: The text message to display. <br> - `$type`: The type of message (e.g., "ok", "warning", "error", "info").
`reset` Clears all entered messages in the event box and resets the skip flag. None
`skip` Sets the skip flag to true, which prevents the event box from being displayed on the next `show` call. None
`noskip` Sets the skip flag to false, allowing the event box to be displayed on the next `show` call. None
`show` Displays the event box with added messages. - `$closebutton` (optional): Indicates whether to include a close button for each message (defaults to `false`).
