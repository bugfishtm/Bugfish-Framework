x_class_block

Introducing the "x_class_block" PHP session security block class: Effectively prevents actions through session counters. This class exclusively operates within sessions, supporting time-based blocks that can be lifted after a designated duration if specified during construction. Leveraging PHP sessions, this class operates without the need for MySQL tables. Enhance security seamlessly.

The x_class_block class essentially offers a mechanism to control actions based on session counters and optionally time-based blocking. It helps manage access to certain functionalities based on session activity. It's important to note that if the $dummy parameter is set to true, the class instance doesn't perform any actions, useful for creating a placeholder instance.

Function Description
__construct($pre_key, $maxcount, $block_time = false, $dummy = false) Initializes the class instance. It sets up the session key, maximum allowed count before blocking, and an optional time period for blocking. If $dummy is set to true, it creates a dummy instance.
blocked() Checks if the current session is blocked based on the set conditions. If the block time is specified, it checks if the time has expired. If the maximum count is reached, it sets the block state. This function returns true if blocked, false otherwise.
increase($int = 1) Increases the session counter by the specified integer value ($int). This function also calls the blocked function to update the block status.
decrease($int = 1) Decreases the session counter by the specified integer value ($int). This function also calls the blocked function to update the block status.
reset() Resets the session counter to zero and clears the block timestamp if applicable. This function also calls the blocked function to update the block status.
